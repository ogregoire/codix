### Task 3: Create JS plugin with symbol extraction

**Files:**
- Create: `src/plugin/js/mod.rs`
- Modify: `src/plugin/mod.rs` â€” add `pub mod js;` and register in `PluginRegistry::new()`

- [ ] **Step 1: Create the JS plugin module**

Create `src/plugin/js/mod.rs` with:
- `JsPlugin` struct implementing `LanguagePlugin`
- `name()` returns `"js"`
- `display_name()` returns `"JavaScript"`
- `can_handle()` matches `.js`, `.mjs`, `.cjs`, `.jsx`, `.ts`, `.tsx`
- `tree_sitter_language()` returns `tree_sitter_typescript::LANGUAGE_TSX.into()`
- `extract_symbols()` extracts classes, functions, methods, fields, constructors

Symbol extraction approach:
- Walk top-level statements for class declarations and function declarations
- For `function_declaration`: kind `function`, qualified name is the function name
- For `class_declaration`: kind `class`, then extract members from class body
- For `lexical_declaration` / `variable_declaration` with arrow function or function expression as value: kind `function`
- Class members: `method_definition` â†’ kind `method` (or `constructor` if name is `constructor`)
- Class fields: `public_field_definition` â†’ kind `field`
- Visibility: `private` for names starting with `#`, `public` otherwise
- Package: empty string
- Signatures for methods/functions: `name()` (no param types)

- [ ] **Step 2: Register the plugin**

In `src/plugin/mod.rs`, add `pub mod js;` and in `PluginRegistry::new()`:
```rust
registry.register(Box::new(js::JsPlugin));
```
